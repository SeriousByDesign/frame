## Execution order

After all milestone features are written, append an execution order section as the final section of BACKLOG.md.

Ordering rules — apply in priority order:
1. **Milestone sequence** — earlier milestones always execute first
2. **Hard dependencies** — if feature B requires feature A, A comes first; note the dependency inline
3. **Conditional features** — implementation items follow their evaluation item; mark as conditional
4. **User preferences** — any ordering preferences stated in FEATURES override the default within their milestone
5. **Risk-first default** — where no other rule applies, features that could reveal blockers or invalidate assumptions go before features that build on settled ground

```markdown
---

## Execution order

_Last ordered: [date] · Re-run this step if new backlog entries are added._

1. PREFIX-NNN · [Feature name]
2. PREFIX-NNN · [Feature name] — requires PREFIX-NNN
3. PREFIX-NNN · [Feature name] _(conditional — only if PREFIX-NNN decides to implement)_
```

Dependency and conditional notes are the only prose allowed. No commentary, no rationale — just the ordered list.
